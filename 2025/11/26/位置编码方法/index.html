position embedding

发表于 2025-11-26 分类于 LLM Changyan：

位置编码方法

  1. 正弦，余弦编码方式：

$PE(pos, 2i) = \sin\left(\frac{pos}{10000^{2i/d}}\right)$

$PE(pos, 2i+1) = \cos\left(\frac{pos}{10000^{2i/d}}\right)$

其中，pos是token在句子序列中的位置，d表示位置编码的总维度，2i 和2i+1 分别表示该pos的偶数和奇数维度。

对于某个pos的某个维度来说，周期是2π⋅${10000^{2i/d}}$，比如固定维度为0维，那么pos的周期就是2π，但是每个维度的周期又是递增的，所以理论上来说这个位置编码方式能表示比较多的位置。

属于绝对位置编码，对token之间的相对位置捕捉不够直接

  1. 可学习的绝对位置编码

    类似于对token的embedding化那样，弄一个权重矩阵来学习每个位置的向量化，这样的化句子的长度一定是固定的，还可能过拟合。

  2. 旋转位置编码RoPE

属于相对位置编码，一般相对位置编码通过修改注意力机制来引入相对位置信息。目标是找到一种方法，使得查询（query）和键（key）的内积能够仅依赖于词嵌入和它们的相对位置，即：

$<f_q(x_m,m),f_k(x_n,n)> = g(s_m,x_n,m-n)$

其中$f_q(x_m,m)$,$f_k(x_n,n)$分别表示处于m位置的$x_m$token向量对应的q向量和处于n位置的$x_n$token向量对应的k向量,< , >则表示点积。于是，作者在复数知识上找到对应的方法，先假定词嵌入的维度为2维，则满足如下等式：

$f_q(x_m,m)=(W_qx_m)e^(imθ),f_k(x_n,n)=(W_kx_n)e^(inθ)$

具体推导过程可以参考一文通透位置编码：从标准位置编码、复数、欧拉公式到旋转位置编码RoPE(含其推导与代码实现)-CSDN博客，此时用于点积计算的q向量是先通过$W_q$和词向量乘积得到q向量再乘以旋转矩阵，同理k向量也是这么算的。

由于先用的2维推导的，扩展成更高维度后，作者发现也可以，但还是两两为一组，即对于同一个pos邻近的两维共享一个角度$theta$。整体流程是：

  • 对于 token 序列中的每个词嵌入向量，首先计算其对应的 query 和 key 向量
  • 然后对每个 token 位置都计算对应的旋转位置编码
  • 接着对每个 token 位置的 query 和 key 向量的元素按照 两两一组 应用旋转变换
  • 最后再计算 query 和 key 之间的内积得到 self-attention 的计算结果

该方法的优势是：不仅能够灵活处理不同长度的序列，还能够随着相对距离的增加而衰减词之间的依赖关系，能够更好地捕捉词之间的依赖关系。
